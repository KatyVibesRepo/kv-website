# Public Launch Plan

## Development

- Build and test locally on the Mac.
- Use Stripe test mode.
- Create real event examples.
- Confirm ticket/table inventory logic.
- Confirm reservation records appear in admin.
- Confirm check-in works at the door.

## Pre-launch

- Add admin login protection.
- Move the database to hosted Postgres.
- Configure Stripe live keys.
- Configure real email confirmations.
- Finalize refund, no-show, and table-release policy.
- Test with one private event.

## Launch

- Deploy the app to the production domain.
- Point public event and reservation links to the new event pages.
- Monitor orders, reservations, Stripe payments, and check-ins closely for the first few events.
